## How it works

<Steps>
  <Step title="Download a skill">
    Download individual `.skill` files from the `skills/` directory on GitHub, or grab `robotics-skills-suite-bundle-76.zip` for all 76 at once.
  </Step>
  <Step title="Install in Claude">
    Click **Save skill** in Claude Desktop or Cowork to install the `.skill` file. Each skill declares its triggering description in its frontmatter.
  </Step>
  <Step title="Trigger with natural language">
    Ask Claude to run the skill using natural language. For example: *"Run an ISO 12100 risk assessment on a new cobot palletizing cell."*
  </Step>
  <Step title="Receive audit-ready xlsx output">
    The builder skill produces a multi-tab xlsx workbook. Run the matching reviewer skill to get a visual dashboard with KPI tiles, compliance charts, and findings.
  </Step>
</Steps>

## The builder + reviewer pattern

Every deliverable in the suite follows the same pattern: a **builder** skill produces a structured xlsx artifact, and a **reviewer** skill runs a confirmation-measures checklist against it. Reviewer outputs include a visual dashboard with KPI tiles, a pie chart, a compliance bar, a stacked rating breakdown by section, and a findings table with recommended actions.

<Note>
  Skills hand off via stable xlsx contracts. A change upstream — for example, updating a hazard severity in the ISO 12100 Risk Assessment — automatically flows to every downstream skill that consumes it: PLr determination, compliance matrix, and Declaration of Conformity.
</Note>
