The Robotics Skills Suite is a curated set of 38 builder skills and 38 matching reviewer skills that automate every structured xlsx deliverable in the modern industrial-robotics lifecycle. Every phase that historically required hours of hand-editing in Excel now has a builder that produces an audit-ready, multi-tab xlsx workbook and a reviewer that confirms it against the relevant standard. Skills hand off via stable xlsx contracts, so a change upstream in the Risk Assessment automatically flows to every downstream phase — PLr determination, compliance matrix, and Declaration of Conformity — without manual copy-paste.

## The builder + reviewer pattern

Every deliverable in the suite follows the same two-skill pattern. The **builder** skill takes a JSON input (plus the upstream artifact when chaining) and produces a structured multi-tab xlsx workbook with NAVY/Calibri styling and PLr/SIL color coding. The **reviewer** skill runs a confirmation-measures checklist over the builder's output and produces a separate visual dashboard — it never modifies the source artifact.

<Note>
  Because reviewers never modify source artifacts, you can run a reviewer at any time to get an independent confirmation-measures view — even on artifacts produced outside the suite.
</Note>

## Who this is for

The suite is designed for the engineers and teams who ship real robot cells and autonomous systems:

- **Robot integrators** producing ISO 10218-2 / ANSI R15.06 compliance documentation for every cell delivery
- **OEM machine builders** generating CE marking and ANSI safety files at scale
- **Cobot deployers** running ISO/TS 15066 biomechanical validation, SSM, and PFL plans
- **AMR / AGV operators** executing ISO 3691-4 risk assessments and fleet manager architectures
- **ROS2 development teams** producing system architecture, URDF, behavior tree, Nav2, and TF spec documents
- **OT cybersecurity teams** running IEC 62443-3-2 risk assessments and zone and conduit segmentation
- **AI/ML in robotics teams** producing datasheets for datasets, model cards, and perception test catalogs

## The chain compounding mechanism

The suite's core advantage is that skills hand off via stable xlsx contracts. Every builder produces an xlsx workbook with named, versioned tabs. Downstream builders read from those named tabs as their input — so when you update a hazard severity in the ISO 12100 Risk Assessment, the new risk number flows automatically to the PLr determination, the compliance matrix, and the Declaration of Conformity.

This chain runs across eight lanes — Foundation, Cobot, AMR, Cell+Ops, ROS2, V&V, AI/ML, and Cybersecurity — and is documented in detail in [The Chain](/the-chain).

<Tip>
  Start with the Foundation lane. The ISO 12100 Risk Assessment xlsx is consumed by five downstream skills. Getting that workbook right first pays dividends across the entire project.
</Tip>
