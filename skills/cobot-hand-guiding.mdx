The cobot hand-guiding device specification builder produces a structured xlsx workbook defining all hardware and functional requirements for a hand-guiding device that enables an operator to directly guide a cobot through programmed or collaborative paths. Hand-guiding (HG) is the fourth collaborative operation mode defined in ISO/TS 15066:2016, and it requires a specific enabling device design — typically a 3-position enabling switch — along with guiding speed limits, force sensing thresholds, and activation logic that prevents unintended robot motion.

    **3-position enabling switch design**

    The enabling switch is the primary safety control element for hand-guiding. ISO/TS 15066 and IEC 60947-5-8 define the three-position requirement:

    <CardGroup cols={2}>
      <Card title="Position 1 — Inactive" icon="circle-xmark">
        Switch at rest (not pressed). Robot motion is inhibited. Transition from Position 1 to Position 2 requires a deliberate grip action by the operator.
      </Card>
      <Card title="Position 2 — Enabled" icon="circle-check">
        Switch pressed to mid-travel (normal operating grip). Hand-guiding motion is enabled. Robot follows the operator's applied force within the guiding speed limit.
      </Card>
      <Card title="Position 3 — Panic" icon="circle-exclamation">
        Switch pressed through to full travel (panic grip, beyond normal range). Robot motion is immediately inhibited — equivalent to a safe stop. Prevents freeze-response from amplifying contact force.
      </Card>
      <Card title="Transition monitoring" icon="arrows-left-right">
        The safety controller must monitor transitions between all three positions and detect switch faults (stuck in Position 2, transition skip) as a safety function.
      </Card>
    </CardGroup>

    **Guiding speed limit**

    ```text
    ISO/TS 15066:2016 default maximum guiding speed:  ≤ 250 mm/s (TCP)

    Application-specific limits:
      - Precision path teaching:           ≤ 50 mm/s
      - General collaborative guiding:     ≤ 250 mm/s
      - Gravity-compensated free motion:   ≤ 250 mm/s (with dynamic monitoring)

    Speed limit enforcement: safety-rated speed monitor (SLS function)
    Safety integrity:        PLd / SIL 2 minimum (per ISO 10218-2:2025)
    ```

## Standard reference

**ISO/TS 15066:2016, Section 5.6** — *Hand-guiding (HG)*

Hand-guiding is defined as a mode where a human manually guides a robot that operates under power, using a hand-guiding device that includes an enabling device. The standard requires the enabling device to comply with IEC 60947-5-8 (three-position enabling devices) and the guiding speed to be limited to a safe level verified against the biomechanical data.

<Info>
  ISO 10218-2:2025 (the full industrial robot installation standard) references hand-guiding as a collaborative operation mode and requires the enabling function to be implemented as a safety function with integrity appropriate to the risk assessment outcome — typically PLd / SIL 2.
</Info>

## Enabling switch selection criteria

| Criteria | Requirement | Notes |
|---|---|---|
| Position count | Three-position per IEC 60947-5-8 | Two-position enabling switches do not satisfy the panic-grip requirement |
| Safety rating | PLd / SIL 2 minimum | Higher integrity may be required by site-specific risk assessment |
| Grip geometry | Ergonomic for sustained hand-guiding | Fatigue must not cause unintentional transition to Position 3 |
| Cable entry | IP54 minimum | Industrial environment ingress protection |
| Certification | TÜV, UL, or equivalent | Required for machinery CE marking file |

<Tip>
  Select an enabling switch with a clearly distinguishable tactile difference between Position 2 (enabled) and Position 3 (panic), but not so large a force step that an operator in a panic response cannot naturally reach Position 3. This ergonomic balance is essential for the panic-grip function to work as intended.
</Tip>

<Warning>
  A standard two-position deadman switch does **not** satisfy the ISO/TS 15066 / IEC 60947-5-8 three-position enabling requirement. Using a two-position switch is a critical non-conformance that will be flagged in the reviewer dashboard as a Stop-shipment finding.
</Warning>
