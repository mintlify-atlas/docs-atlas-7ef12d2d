The Speed and Separation Monitoring plan builder produces a structured xlsx workbook specifying how a collaborative robot cell maintains a protective separation distance between the robot and any person in the collaborative workspace. The plan implements the SSM safeguarding mode defined in ISO/TS 15066:2016, calculating the minimum protective separation distance (Sp) using the standardised formula, specifying the detection system that enforces it, and documenting the full response time budget from detection to robot stop.

    **Core Sp formula**

    The protective separation distance is calculated as:

    ```text
    Sp = (vc + vh + vs) × (Tr + Ts) + Zs + Zd + Zr

    Where:
      vc  = maximum robot TCP speed at the relevant speed step (mm/s)
      vh  = maximum human walking speed (typically 1600 mm/s per ISO/TS 15066)
      vs  = speed measurement uncertainty of the detection system (mm/s)
      Tr  = robot stop time from detection signal to full stop (s)
      Ts  = detection system response time (s)
      Zs  = position uncertainty of the safety system (mm)
      Zd  = intrusion depth before detection (mm)
      Zr  = position uncertainty of the robot (mm)
    ```

## Standard reference

**ISO/TS 15066:2016, Section 5.5** — *Speed and Separation Monitoring (SSM)*

SSM is one of four collaborative operation modes defined in ISO/TS 15066. The standard requires that the protective separation distance be maintained at all times; if the separation distance is violated, the robot must stop or reduce speed before the human can reach the collaboration boundary.

<Info>
  The human speed parameter vh defaults to 1600 mm/s in ISO/TS 15066 Annex B. A lower value may be used only if justified by a site-specific study with documented evidence — this justification is subject to reviewer scrutiny.
</Info>

## Sp sensitivity analysis

Understanding which parameters dominate the Sp budget helps designers optimise the system:

| Parameter | Typical contribution | Design lever |
|---|---|---|
| `vc` (robot speed) | 30–50% | Reduce max speed in inner zone |
| `vh` (human speed) | 20–40% | Fixed at 1600 mm/s unless justified |
| `Tr` (robot stop time) | 15–30% | Select robot with certified fast-stop |
| `Ts` (sensor response) | 5–15% | Choose low-latency safety scanner |
| `Zs + Zd + Zr` (position uncertainty) | 5–20% | Reduce sensor uncertainty, improve calibration |

<Tip>
  The largest gains in Sp reduction (allowing the robot to work closer to humans at higher speed) typically come from reducing Tr through robot selection or brake design — not from reducing vh or sensor uncertainty.
</Tip>

<Warning>
  SSM does not eliminate contact risk — it maintains separation to prevent contact. If contact is possible in any operational mode (including fault conditions), the PFL safeguarding mode analysis must also be performed.
</Warning>
